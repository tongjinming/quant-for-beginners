## 本章目标

- 理解回测的含义：用历史数据，假装过去按规则交易了一遍
- 学会把策略信号转换成实际仓位 `position`
- 掌握 `signal.shift(1)` 避免未来函数
- 计算策略日收益、买入持有收益、大盘基准收益
- 画出策略净值曲线，并用胜率、最大回撤初步评价策略

## 关键概念

### 1. 回测不是预测

回测回答的是：

> 如果过去真的按这套规则交易，账户曲线会长什么样？

但它不能证明未来一定赚钱。历史表现好，可能只是刚好适配了过去那段行情；市场环境变化后，策略可能失效。

一句话记忆：**回测是体检，不是算命。**

### 2. 信号与仓位

第三章的策略信号是：

```python
signal = (MA5 > MA20).astype(int)
```

含义：

| 值 | 含义 |
|---|---|
| 1 | 短期趋势强于长期趋势，理论上持有 |
| 0 | 短期趋势弱于长期趋势，理论上空仓 |

但回测不能当天收盘算完信号后，又假装当天已经成交，所以要推迟一天：

```python
position = signal.shift(1).fillna(0).astype(int)
```

这代表：**今天收盘产生的信号，明天才执行。**

### 3. 策略收益

股票日收益：

```python
ret = Close.pct_change()
```

策略日收益：

```python
strategy_ret = position * ret
```

含义：

- 持仓时：股票涨跌算到策略上
- 空仓时：股票涨跌与策略无关，收益为 0

累计净值：

```python
nav_strategy = (1 + strategy_ret).cumprod()
```

从 1 元钱开始，每天收益连乘，得到净值曲线。

### 4. 基准对比

只看策略收益没有意义，要和基准比：

| 曲线 | 作用 |
|---|---|
| 双均线策略 | 检查规则交易的结果 |
| 买入持有标的 | 判断策略有没有跑赢“什么都不做” |
| 买入持有 SPY | 判断策略有没有跑赢大盘 |

### 5. 胜率

胜率统计的是一轮完整的“买入 → 卖出”：

```text
胜率 = 盈利交易次数 / 完整交易次数
```

注意：高胜率不等于高收益。因为一次大亏可能抵消很多次小赚。

### 6. 最大回撤

最大回撤衡量的是：

> 净值从历史最高点跌到后续低点，最惨时跌了多少。

注意：最高点到最低点之间**不要求单调下降**，中间可以反弹、横盘、再下跌。最大回撤只关心“先出现的高点”和“之后出现的低点”之间，哪一组跌幅最大。

数学表达：

假设净值序列为：

```text
f(1), f(2), ..., f(T)
```

其中 `f(t)` 表示第 `t` 天的净值。

第 `t` 天之前见过的历史最高净值是：

```text
peak(t) = max{ f(1), f(2), ..., f(t) }
```

第 `t` 天的回撤是：

```text
drawdown(t) = f(t) / peak(t) - 1
```

因为 `f(t)` 不会高于 `peak(t)`，所以 `drawdown(t)` 通常是 0 或负数。

最大回撤是所有日回撤中最小的那个值：

```text
max_drawdown = min{ drawdown(t) }
```

如果想用正数表示“最大跌幅”，也可以写成：

```text
最大回撤幅度 = max{ 1 - f(y) / f(x) }
```

其中要求：

```text
x < y
```

也就是：`x` 是前面的某个高点时间，`y` 是后面的某个低点时间。

更直观地说：

```text
最大回撤 = 找一个前面的高点，再找它后面的低点，让“跌幅百分比”最大
```

代码：

```python
peak = nav_series.cummax()
drawdown = nav_series / peak - 1
max_drawdown = drawdown.min()
```

最大回撤越深，说明历史上策略曾经承受过更大的心理压力。

## 代码逐段注释

### 1. 环境准备与数据源切换

第四章原本使用 `yfinance`，但运行时遇到：

```text
YFRateLimitError('Too Many Requests. Rate limited. Try after a while.')
```

解决方案参考 `ch03_移动平均线策略.md`，改用 `akshare` 新浪源：

```python
import akshare as ak
import os

for k in ['HTTP_PROXY', 'HTTPS_PROXY', 'http_proxy', 'https_proxy']:
    os.environ.pop(k, None)
os.environ['NO_PROXY'] = '*'

raw = ak.stock_us_daily(symbol=TICKER, adjust='qfq')
```

关键处理：

```python
raw['date'] = pd.to_datetime(raw['date'])
raw = raw.set_index('date').sort_index()
raw.columns = [c.capitalize() for c in raw.columns]
df = raw[['Close']].tail(PERIOD).copy()
```

注意：

- `akshare` 返回列名是小写，需要统一成首字母大写
- `PERIOD` 改成交易日数量，例如 `500` 表示最近约 2 年
- 清空代理变量是为了避免代理影响国内数据源连接

### 2. 生成双均线信号

```python
df['MA5'] = df['Close'].rolling(5).mean()
df['MA20'] = df['Close'].rolling(20).mean()
df['signal'] = (df['MA5'] > df['MA20']).astype(int)
```

核心规则：

```text
MA5 > MA20：持有
MA5 <= MA20：空仓
```

### 3. 避免未来函数

```python
df['position'] = df['signal'].shift(1).fillna(0).astype(int)
```

这是本章最重要的一行。

如果不 `shift(1)`，就等于“今天收盘后才知道的信号，却假装今天已经按这个信号交易”，这会偷看未来。

### 4. 标记买卖动作

```python
df['position_change'] = df['position'].diff().fillna(0)
df['action'] = ''
df.loc[df['position_change'] > 0, 'action'] = '买入'
df.loc[df['position_change'] < 0, 'action'] = '卖出'
```

解释：

| 仓位变化 | 动作 |
|---|---|
| 0 → 1 | 买入 |
| 1 → 0 | 卖出 |
| 不变 | 无操作 |

### 5. 计算净值曲线

```python
df['ret'] = df['Close'].pct_change().fillna(0)
df['strategy_ret'] = df['position'] * df['ret']
df['buyhold_ret'] = df['ret']
```

大盘基准也改用 `akshare`：

```python
spy_raw = ak.stock_us_daily(symbol=BENCHMARK, adjust='qfq')
spy = spy_raw[['Close']].tail(PERIOD).copy()
spy.columns = ['SPY_Close']
df = df.join(spy, how='inner')
```

最后计算三条净值：

```python
df['nav_strategy'] = (1 + df['strategy_ret']).cumprod()
df['nav_buyhold'] = (1 + df['buyhold_ret']).cumprod()
df['nav_market'] = (1 + df['market_ret']).cumprod()
```

### 任务 3：用三句话解释“回测是什么”

1. 回测就是拿历史行情数据，假装自己过去真的按某套规则买入、持仓、卖出了一遍。
2. 它能告诉我们这套规则过去赚没赚钱、亏得最惨时有多深、有没有跑赢买入持有和大盘。
3. 但回测不是预测未来，过去表现好只能说明值得进一步研究，不能保证以后一定赚钱。

## 本章踩坑记录

### 1. `yfinance` 限流

现象：

```text
YFRateLimitError('Too Many Requests. Rate limited. Try after a while.')
```

影响：

- 下载到空数据
- `len(df) = 0`
- 买入、卖出次数全是 0
- 后续收益、回撤都无法正常计算

解决：

- 参考第 3 章，改用 `akshare.stock_us_daily()`
- 清理代理环境变量
- 将 `PERIOD='2y'` 改为 `PERIOD=500`

### 2. pandas `.last()` 方法兼容问题

原写法：

```python
recent = df.last('12M')
```

新版 pandas 可能不再支持。

改法：

```python
recent = df.tail(250)
```

对教学场景足够直观：近 250 个交易日约等于近 12 个月。
